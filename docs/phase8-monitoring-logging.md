# Phase 8: Monitoring & Logging

## Overview
Monitoring and logging were implemented to track system activity, detect issues, and identify potential security threats.

## CloudWatch Logging
- Enabled AWS CloudWatch for collecting logs
- Used it to monitor Kubernetes cluster activity
- Helps track errors and system events

## GuardDuty
- Enabled AWS GuardDuty
- Detects suspicious activities such as unauthorized access
- Provides security alerts

## Prometheus & Grafana
- Can be used for metrics and visualization
- Helps monitor CPU, memory, and application performance

## Why Monitoring is Important
Monitoring allows early detection of issues and helps respond quickly to security threats.
